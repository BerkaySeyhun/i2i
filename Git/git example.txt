Scenario:
Our product’s r2.5.4 version is being used in production and we have r2.5.5 that will be released in the future. 
Also there is r3.0 version which is in the development process. Suddenly an error occured in r2.5.4 and we are creating a branch called r2.5.4.1 Bugfix in order to solve this.
Then we add those changes to our future branch and development branch.

Objectives : 
•	To learn how to use git.
•	To understand importance of version control.
•	To carry out a real world scenario

Exercise Keywords: git, version control, merge, branch.
